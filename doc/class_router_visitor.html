Detailed Description

Base class for router configuration visitors.

RouterVisitor objects are used to traverse the router configuration graph. They are usually passed to the Router::visit_downstream() and Router::visit_upstream() functions.

true to continue traversing, false to stop

A Router configuration traversal calls this function once on each reached port. Configuration traversal is breadth-first; distance is the minimum connection distance from the traversal's start point to e/port. A traversal will call a port's visit() function at most once. from_e and from_port specify one port that is connected to e and port (there may be more than one). If isoutput is true, then e/port is an output port and from_e/from_port is an input port; if isoutput is false, the opposite holds.

The return value specifies whether traversal should continue through the e element. If it is true, then the traversal will continue based on e's packet flow specifier, via ports that connect to e/port (see Element::flow_code() for more).

The default implementation always returns true.

Reimplemented in ElementNeighborhoodTracker, and ElementCastTracker.
