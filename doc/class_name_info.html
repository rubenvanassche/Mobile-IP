Detailed Description

Manages Click name databases.

NameInfo is the user interface to Click's name databases, a general mechanism for mapping names to values. NameInfo and NameDB implement generally useful properties like names local to a compound element context.

Every name database maps names, which are nonempty strings, to values, which are sequences of bytes. A database is identified by its type, which indicates the kind of name stored in the database, and, optionally, its compound element context, which is a pair of router and element name prefix. The database is used for queries that take place in the given context. A query bubbles up through the databases, starting with the most specific context and getting more general, until it succeeds. A database stores values with a fixed size; it is an error to query a database with the wrong size.

Many common types are defined in the NameDB::DBType enumeration, but NameInfo allows arbitrary other types to be added at run time.

Here's an example of how to query NameInfo:

EtherAddress ethaddr;
if (NameInfo::query(NameInfo::T_ETHERNET_ADDR, this, name, &ethaddr, 6))
click_chatter("Found address!");
else
click_chatter("Didn't find address.");

Many elements also add their own databases to NameInfo; see, for example, the IPNameInfo or IPFilter elements for examples.

Find or create a name database.

Parameters
typedatabase type
contextcompound element context
value_sizesize of values stored in database
createwhether to create a DynamicNameDB if no database exists

Returns an installed name database matching type and context. (If context is non-null, then the database matches the implied router and compound element context. Otherwise, the database is the unique global database for the given type.) If create is true, and no database exists exactly matching type and context, then a DynamicNameDB is created and installed with that type, context, and value_size. Otherwise, the search bubbles up through the prefixes of context until an installed database is found.

Returns null if no installed database is found. value_size must match the value size in the returned database.

Most users will use query() and define() directly, not call getdb().

Install a name database.

Parameters
dbname database
contextcompound element context

Installs the given name database for the compound element context implied by context. (If context is non-null, then the database is installed for the implied router and compound element context. If it is null, the database is installed globally.) The query() and define() operations apply only to installed databases.

It is an error to install a database that has already been installed. It is also an error to install a database for a compound element context that already has a different database installed. An installed NameDB is automatically destroyed when its containing NameInfo is destroyed (for example, when context's router is destroyed).

Query installed databases for name, returning a 32-bit integer value.

Parameters
typedatabase type
contextcompound element context
namename to look up
value_storevalue storage
Returns
true iff the query succeeded

Queries all installed type databases that apply to the compound element context, returning the most specific value matching name. The value is stored in value_store. The installed databases must have a value size of 4.

If no matching name is found, query_int checks whether name unparses into a 32-bit integer value (for example, "30"). If so, *value_store is set to the corresponding integer and true is returned. Otherwise, false is returned.

If no database exists exactly matching that type and context, a new DynamicNameDB is created and installed with those values (and the given value_size). A name might not be defined if the existing database for that type and context doesn't support definitions, or if no new database can be created.
