8 =c

10 AddressInfo(NAME ADDRESS [ADDRESS...], ...)
11 
12 =s information
13 
14 specifies address information
15 
16 =io
17 
18 None
19 
20 =d
21 
22 Lets you use mnemonic names for IPv4 and IPv6 addresses, IPv4 and IPv6
23 address prefixes, and Ethernet addresses. Each argument has the form `NAME
24 ADDRESS [ADDRESS...]', which associates the given ADDRESSes with NAME. For
25 example, if a configuration contains this AddressInfo element,
26 
27  AddressInfo(mauer 10.0.0.1, mazu 10.0.0.10);
28 
29 then other configuration strings can use C<mauer> and C<mazu> as mnemonics
30 for the IP addresses 10.0.0.1 and 10.0.0.10, respectively.
31 
32 The mnemonic names introduced by AddressInfo elements are local with
33 respect to compound elements. That is, names created inside a compound
34 element apply only within that compound element and its subelements. For
35 example:
36 
37  AddressInfo(mauer 10.0.0.1);
38  compound :: {
39  AddressInfo(mazu 10.0.0.10);
40  ... -> IPEncap(6, mauer, mazu) -> ... // OK
41  };
42  ... -> IPEncap(6, mauer, mazu) -> ... // error: `mazu' undefined
43 
44 Any name can be simultaneously associated with an IP address, an IP network
45 address, and an Ethernet address. The kind of address that is returned is
46 generally determined from context. For example:
47 
48  AddressInfo(mauer 10.0.0.1 00-50-BA-85-84-A9);
49  ... -> IPEncap(6, mauer, ...) // as IP address
50  -> EtherEncap(0x0800, mauer, ...) -> ... // as Ethernet address
51  ... -> ARPResponder(mauer) -> ... // as IP address and Ethernet address!
52 
53 An optional suffix makes the context unambiguous. C<NAME> is an ambiguous
54 reference to some address, but C<NAME:ip> is always an IPv4 address,
55 C<NAME:ipnet> is always an IPv4 network address (IPv4 address prefix),
56 C<NAME:ip6> is always an IPv6 address, C<NAME:ip6net> is always an IPv6
57 network address, and C<NAME:eth> is always an Ethernet address.
58 
59 Names with both address and prefix definitions are preferentially parsed as
60 addresses. For example:
61 
62  AddressInfo(boojum 10.0.0.1/24); // defines address and prefix
63  a1 :: ARPResponder(boojum 00-01-02-03-04-05);
64  // a1 will have the same configuration as:
65  a2 :: ARPResponder(10.0.0.1/32 00-01-02-03-04-05);
66 
67 To prefer the network prefix, use C<NAME:ipnet>.
68 
69 If C<NAME:ipnet> is a valid IPv4 network address, then C<NAME:bcast> is a
70 valid IPv4 address equaling the broadcast address for that network. This is
71 obtained by setting all the host bits in the address prefix to 1.
72 
73 C<NAME:gw> usually equals the default gateway address for network
74 C<NAME>. If C<NAME> is a device, then C<NAME:gw> can sometimes be
75 looked up explicitly; otherwise, C<NAME:gw> is calculated from
76 C<NAME:ipnet> by setting the lowest-order bit in the network address.
77 
78 =head1 DEFAULT ADDRESSES
79 
80 If you do not define an address for a given NAME, AddressInfo will use the
81 default, if any. Defaults are as follows:
82 
83 =over 2
84 
85 =item *
86 
87 For Ethernet addresses, Click checks for an Ethernet device named NAME, and
88 uses its address. (At userlevel, this works only on BSD and Linux.)
89 
90 =item *
91 
92 For IPv4 addresses, Click checks for a network device named NAME, and uses
93 its primary IPv4 address.
94 
95 =item *
96 
97 For C<NAME:gw>, Click checks for a network device named NAME, and uses its
98 default gateway.
99 
100 =back
101 
102 These defaults are not available on all platforms.
103 
104 =a
